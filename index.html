This week I worked on my first program for this class! A hello world in a Java file. Where I learned to make a Java file and set it up, as well write a line in the console. I also learned about variables and that there are two types of variables in Java. You have Primitive variables and Non-primitive variables. The Primitive variables are byte, short, int, long, float, double, boolean and char. 

// Byte can store whole numbers between -128 and 127
byte myAge = 16;
// Short can store whole numbers between -32768 and 32767
short shoeSize = -23754;
// Int can store whole numbers between -2147483648 and 2147483647
int amountPeople = 93564789;
// Long can store whole numbers between -9223372036854775808 and 9223372036854775807
long money = 8374736746344393;
// Float can store fractional numbers, is great for storing numbers to the 6th or 7th decimal point
float timer = 5.637348;
// Double can store fractional numbers, is great for storing numbers to the 15th decimal point
double score = 183.38473883734292;
// Boolean stores a true or false value
boolean alive = false;
// Char stores a single character/letter or ASCII values
char myLetter = "L";

The Non-primitive variable types are string, array and classes. I don't learn these till a later chapter so I have nothing to show for them.

This week I learned about the way to change the direction of an Imperative programming language. Also known as flow controll. The first of many ways to change the flow of an Imperative program is with an if/else statement. The if part of the if/else statement always comes first, and will have some conditions it has to follow to run. 

/* This the if statement, everything in the () is the conditions to make this if statement work.
There can be more than one condition, just add && to say and, or || to say or. */
if (i == 5) {
    System.out.println("Hello world");
}

// This is an if statemnt with more than one condition
if (x == 22 && y == 10 || v == 3) {
    System.out.println("Goodbye earth");
}

Now there is also the else part which you do not need, but can be added on for another route for the code. Like if you had a double variable called num get a random number between 1 and 10, and a if/else statement that says if (num <= 5) say "little numbers", else say "big numbers". You can also add an if behind an else for more descrition. Like else if (num == 10) say "huge number". 

/* This is a demonstration of an if/else statement
It also has an else if as well */

// This is the variable, it is a random number
double num = Math.floor(Math.random() * (10 - 1 + 1) + 1);

// This will run if its condition is met
if (num <= 5) {
    System.out.println("little numbers");
}
// This will run if its condition is met and the code above did not run
else if (num == 10) {
    System.out.println("huge number");
}
// This will run if the code above it does not run
else {
    System.out.println("big numbers");
}

Another thing I learned was switch statements. These are pretty cool little things that help out when programming a Java file. You could have it say if (up), go_down(), which can also be written as direction == up, go_down(), and you can add as many of these as you need. 

// Format them with a comma at the end
if (up), go_down(),
if (down), go_up();

// It can also be written this way
direction == up, go_down(),
direction == down, go_up(),

The last thing I learned is loops! There is for loops and while loops which both run of a variable and repeat there code a certain amount of times. A for loop has a variable inside of it that if this varible is does not meet the condition will change by either adding or losing numbers until it meets the condition while also running the code in itself. 

// This for loop will run and print what number it's on till it reaches 10
for (var i = 0; i <= 10; i++) {
    System.out.println(i);
}

/* This while loop will run and print what number it's on till it reaches 0
This variable is its number */
byte x = 10;

while (x >= 0) {
    System.out.println(x);
    x--;
}

Now a while loop is very simular to a for loop with a few key diffences. First off it does not have the variable inside it, the variable it has is outside of the loop entirely. Secondly when it changes the value of it's variable it can be anywhere inside the loop. Which is gonna be very helpful in the future.
